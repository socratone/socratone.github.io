#### 고향사랑기부 홈페이지 성능 최적화

새로 출시된 고향사랑기부 홈페이지를 사용하는 중 체감적으로 너무 느리다고 판단하여
주도적으로 문제를 파악해보니 HTML 문서 사이즈가 1MB를 넘어 상급자께 성능 개선 제안을 했습니다.

HTML을 분석해보니 스와이퍼에 리스트로 불러오는 아이템의 수가 너무 많고 보이지 않는 부분까지 렌더링 하여 HTML 텍스트가 보이지 않는 페이지만큼 더 그려지는 문제였습니다.

SVG를 CDN 리소스에서 불러오도록 처리하여 한 번 불러오면 캐싱되고 HTML 텍스트 크기를 1/10까지 감소시켰습니다.

#### 웰로비즈 회원가입 프로세스 개선

회원 가입 프로세스 중 중도 이탈자가 유의미한 지표로 발생하였습니다.
기업 맞춤형 공고 추천을 위해 회원 가입 단계에서 많은 정보를 요구하고 있었고,
이로 인해 사용자 피로도가 증가하여 이탈이 발생한 것으로 파악되었습니다.

필수 입력 값을 최소화하고 일부 항목을 옵셔널로 전환하였습니다.
또한, 회원 가입 완료 이후에도 추가 정보를 자연스럽게 입력할 수 있도록 툴팁, 가이드 UI 등을 활용해 입력을 유도하는 구조로 개선하여 데이터 수집률을 유지하면서 가입 장벽을 낮췄습니다.

기존에도 중도 이탈 후 재접속 시 이전 단계로 복귀하는 기능은 존재했으나,
리다이렉트 로직이 여러 곳에 분산되어 있고 문서화가 되어 있지 않아 동작을 파악하기 어렵고 의도치 않은 이동이 발생하는 문제가 있었습니다.

이를 개선하기 위해 어떤 상태에서 어떤 페이지로 이동해야 하는지 기준을 명확히 정의하고 jsdocs로 문서화하였으며,
불필요한 리다이렉트 로직을 제거하고 AuthGuard 및 로그인 시점에만 동작하도록 구조를 단순화했습니다.

#### 공통 CI/CD 파이프라인 개선

엘리스에서 운영 중인 7개의 Next.js 애플리케이션을 유지보수하는 과정에서,
배포 시마다 약 10분 이상의 시간이 소요되어 개발 및 운영 효율이 저하되는 문제가 있었습니다.
GitLab Runner 기반의 CI/CD 파이프라인이 구성되어 있었으나,
배포 시간이 길어지는 원인에 대한 명확한 분석이 이루어지지 않은 상태였습니다.

이를 개선하기 위해 GitLab CI/CD 문서를 기반으로 기존 파이프라인 구조를 분석하고,
각 stage의 역할과 실행 흐름을 점검했습니다.
그 결과, 테스트 stage에서 이미 build를 수행하고 있음에도 불구하고
build stage에서 동일한 작업을 중복 수행하고 있는 비효율을 확인했습니다.

중복된 build 과정을 제거하고 단일 build로 통합하여 파이프라인을 재구성함으로써,
불필요한 실행 시간을 줄이고 배포 프로세스를 간소화했습니다.

그 결과, 전체 CI/CD 배포 소요 시간이 크게 단축되어 개발 생산성과 배포 효율이 향상되었습니다.

- CI/CD 배포 시간 약 50% 단축 (약 10분 → 5분)

외부 접근이 가능한 환경을 구성했으며, Certbot을 활용해 SSL 인증서를 적용했습니다.
또한 GitHub Actions와 Docker Compose를 이용해
Nginx + Next.js 서버를 자동 배포하는 CI/CD 파이프라인을 구축하여 운영 효율을 높였습니다.

서비스 측면에서는 모바일 사용성을 고려한 UI(다크모드, 폰트 크기 조절 등)를 구현하고,
카카오 OAuth 로그인 및 세션 기반 인증을 적용했습니다.
MongoDB Atlas를 활용해 데이터베이스를 설계하고 API를 구현하여
묵상 콘텐츠 조회 및 사용자 간 공유 기능을 지원했습니다.

- 홈 서버 구축 및 운영 경험 확보 (네트워크, 배포, 보안 설정)
- CI/CD 자동화로 배포 및 운영 효율 개선
- 모바일 UX 중심 SNS 서비스 설계 및 풀스택 개발
