#### 고향사랑기부 홈페이지 성능 최적화

새로 출시된 고향사랑기부 홈페이지를 사용하는 중 체감적으로 너무 느리다고 판단하여
주도적으로 문제를 파악해보니 HTML 문서 사이즈가 1MB를 넘어 상급자께 성능 개선 제안을 했습니다.

HTML을 분석해보니 스와이퍼에 리스트로 불러오는 아이템의 수가 너무 많고 보이지 않는 부분까지 렌더링 하여 HTML 텍스트가 보이지 않는 페이지만큼 더 그려지는 문제였습니다.
또한 각 아이템에 별 SVG가 포함되어 있어 SVG 자체의 HTML 텍스트 양도 많았습니다.

SVG를 CDN 리소스에서 불러오도록 처리하여 한 번 불러오면 캐싱되고 HTML 텍스트 크기를 1/10까지 감소시켰습니다.

- 라이트하우스 퍼포먼스 점수 +10, LCP -1초

#### GTM 이벤트 트래킹

고향사랑기부 서비스 런칭 이후 사용자의 행동 퍼널을 상세하게 분석하여 데이터를 기반으로 기능을 구현하려는 목적으로 시작했습니다.

- 각 이벤트 코드 추가시 단 한 줄만 추가하면서도 퍼널별 20종 이상의 GTM 이벤트 추가됨

#### 공통 CI/CD 파이프라인 개선

엘리스의 7가지 nextjs 앱을 유지보수하며 배포할 때마다 10분 가까운 시간을 소요했습니다.\
gitlab의 Runner를 이용한 ci/cd 시스템이었습니다.\
gitlab ci/cd 문서를 읽고 기존에 작성되어 있던 코드를 파악하고 개선할 수 있는 지점이 있는지 확인하였습니다.\
테스트 stage에서 build 테스트를 한 뒤 build 스테이지에서 한 번 더 build를 하도록 설정되어 있어 불필요한 시간이 더 걸리는 게 문제였습니다.\
build를 한 번만 하도록 수정하여 ci/cd 배포 프로세스 시간이 2배 더 빨라지게 되었습니다.
